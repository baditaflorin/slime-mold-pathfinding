# Postmortem

## What Was Built

V1 is a Mode B static GitHub Pages application:

- React/Vite/TypeScript frontend built into `docs/`.
- WebGPU trail diffusion with CPU fallback.
- Physarum-style agent simulation over normalized city graphs.
- Tokyo and Bucharest city fixtures generated by a Go data pipeline.
- Food-source presets and click-to-drop food sources.
- Route extraction biased by slime trail intensity.
- Browser-side GraphViz SVG rendering through lazy WASM.
- Live repository, PayPal, version, and commit links on the page.
- Local hooks, unit tests, linting, Pages preview, and Playwright smoke test.

## Was Mode B Correct?

Yes. Mode A would have forced map extraction into the browser, which is the wrong place for libosmscout-scale data preparation. Mode C would have added a runtime server without any v1 need for auth, writes, secrets, or cross-device sync.

## What Worked

- GitHub Pages from `/docs` stayed simple.
- GraphViz WASM lazy loading kept the initial payload under the 200KB gzip budget.
- CPU fallback made testing reliable when WebGPU is unavailable.
- The Go generator gave the frontend a stable and validated artifact contract.

## What Did Not Work

- The first smoke harness assumed a fixed local port and collided with an existing process.
- Vitest initially picked up Playwright specs until the test boundaries were narrowed.
- `go test ./...` wandered into `node_modules`, so Go targets now explicitly cover `cmd` and `internal`.

## Surprises

- The installed `govulncheck` binary had to be rebuilt against the current Go version before scanning.
- The generated JSON format intentionally differs from Prettier output, so generated Pages/data artifacts are excluded from Prettier checks.

## Accepted Tech Debt

- The shipped city fixtures are normalized synthetic examples, not full OSM/libosmscout extracts.
- WebGPU currently accelerates trail diffusion/decay, while agent steering remains CPU-side.
- Route optimality is an exploratory heuristic, not a certified transport-planning model.

## Next Improvements

1. Add a real libosmscout export adapter and document the exact extraction command.
2. Move more agent update work into WebGPU buffers to reduce CPU load.
3. Add larger city artifacts through GitHub Releases when fixtures exceed comfortable repo size.

## Time Spent vs Estimate

Estimated: 4 to 6 focused hours for a polished v1 scaffold and live demo.

Actual: one extended implementation pass in this workspace, with most time spent on frontend simulation/rendering and local smoke reliability.
